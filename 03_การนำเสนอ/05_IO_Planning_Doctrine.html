CH.4 OVERVIEW
หลักการพื้นฐาน (อทร.3107 ข้อ ๔-๑): การวางแผน IO ต้องบูรณาการกับกระบวนการวางแผนทางทหารหลักตั้งแต่ต้นจนถึงการปฏิบัติ และต้องดำเนินการควบคู่กับทุกขั้นตอน
โหมดการวางแผน — ๒ ประเภท

๑. การวางแผนในภาพรวม

Overall IO Planning

  • กำหนดวัตถุประสงค์ IO รวม ผลกระทบ TA และ Theme
  • จัดทำ IO Planning Matrix (ภาพที่ ๗)
  • ประสาน/บูรณาการสาขา IO ทั้งหมดให้สอดคล้อง

๒. การวางแผนตามสาขา

Branch-Specific IO Planning

  • วางแผนรายสาขา: PSYOP / MILDEC / OPSEC / EW / CNO
  • แต่ละสาขา ๔ ขั้น: วิเคราะห์ → COA → Wargame → ข้อตกลงใจ
  • ผนวกรายละเอียดเป็น Annex ต่อแผนหลัก

Branch Planning

ขั้น ๑ — กรอบเบื้องต้น (Initial Frame)

  • รวบรวมข้อเท็จจริง IO ปัจจุบัน (IO capabilities)
  • รวบรวมข้อมูลข่าวสารกลุ่มเป้าหมาย (TA intelligence)
  • ประวัติและกระบวนการตัดสินใจของผู้นำ ขตก.
  • ประเมินสภาพแวดล้อมสารสนเทศ (IE Assessment)

ขั้น ๒ — วิเคราะห์ภารกิจ (Mission Analysis)

  • วิเคราะห์เจตนารมณ์ ผบ.: จุดหมาย + วิธีการ + สภาวะสิ้นสุด
  • วิเคราะห์องค์ประกอบยุทธศิลป์: COG → CC → CR → CV
  • กำหนดภารกิจ IO: กิจ + จุดหมาย
  • ตรวจสอบข้อจำกัด IO (Constraint / Restraint)
  • กำหนด หวส. / PIR / EEFI

ขั้น ๓ — กำหนดหนทางปฏิบัติ (COA Dev)

  • กำหนดวัตถุประสงค์ IO + ระดับชั้น + กรอบเวลา
  • รูปแบบ: ผลกระทบ + เป้าหมาย + ความมุ่งหมาย
  • กำหนด MOE สำหรับแต่ละวัตถุประสงค์
  • จัดทำ IO Matrix ฉบับสมบูรณ์
  • ขั้น ๔–๕: IO เข้าร่วม Wargame ปรับปรุงหนทางปฏิบัติ
▶ บ่งการ — Op RESTORE TALOS
  • ขั้น ๑ — Initial Frame: ประเมิน IE ในพื้นที่ TALOS — SIGINT WL, Hybrid Warfare Capabilities, PSYOP Channels
  • ขั้น ๒ — Mission Analysis:
    · COG WL = เจตจำนงรัฐบาล WL + กำลังรบ TALOS
    · EEFI AZ = แผน D-Day, เส้นทางขึ้นบก
    · PIR = ระบบ AD WL, C2 WL ใน TALOS
  • ขั้น ๓ — COA Dev: กำหนด ๔ IO Objectives หลัก (Convince / Deceive / Protect / Disrupt)
  • ขั้น ๔–๕ — Wargame: ทดสอบว่า MILDEC จะหลอก WL ได้ก่อน D-Day หรือไม่
  • ขั้น ๖ — Branch: มอบ PSYOP/MILDEC/OPSEC/EW/CNO วางแผนรายสาขา

IO Objective

มิติของเป้าหมาย — Target Dimensions

  • กายภาพ (Physical) — ระบบ เครื่องมือ สถานีสัญญาณ โครงสร้างพื้นฐาน
  • สารสนเทศ (Information) — ข้อมูล ข่าวสาร กระแสการสื่อสาร
  • จิตวิทยา (Cognitive) — ความคิด ทัศนคติ การรับรู้ ขวัญกำลังใจ

คำเชื่อมความมุ่งหมาย — Purpose Connectors

  • Allow — เพื่ออนุญาต/เปิดโอกาสให้...
  • Cause — เพื่อทำให้เกิด...
  • Create — เพื่อสร้าง...
  • Enable — เพื่อเอื้อให้...
  • Support — เพื่อสนับสนุน...

IO OBJ ๑ — PSYOP

IO OBJ ๒ — MILDEC

IO OBJ ๓ — OPSEC

Protect EEFI ฝ่าย AZ (มิติสารสนเทศ) → Support ความได้เปรียบยุทธการ

IO OBJ ๔ — EW/CNO

Disrupt ระบบ C2 WL ใน TALOS (มิติกายภาพ) → Enable การโจมตี D-Day

Assessment Framework — ภาพที่ ๙ Assessment Levels and Measures
MOE / MOP

MOE — Measurement of Effectiveness

"Are we doing the right things?" — เรากำลังทำสิ่งที่ถูกต้องหรือไม่?

  • วัดระดับผลกระทบต่อวัตถุประสงค์ IO ที่กำหนด
  • ผลกระทบที่ต้องการเกิดขึ้นจริงหรือไม่?
  • ใช้ระดับ ยุทธศาสตร์และยุทธการ
Indicators (สิ่งบอกเหตุ): ข้อมูลเชิงประจักษ์ที่บอกว่า MOE กำลังเป็นไปได้ดี เช่น ผลสำรวจความคิดเห็น, รายงาน SIGINT

MOP — Measurement of Performance

"Are we doing things right?" — เราดำเนินการถูกวิธีหรือไม่?

  • วัดว่ากิจกรรมถูกดำเนินการตามที่กำหนดหรือไม่
  • ใช้ระดับ ยุทธการและยุทธวิธี
  • ตัวอย่าง MOP: จำนวนครั้งออกอากาศ, EEFI ที่ปกปิด, จำนวนการพบปะ KLE

BRANCH PLANNING

PSYOP — ปฏิบัติการจิตวิทยา

  • วิเคราะห์ TA: ผู้ตัดสินใจ ระบบ C2 กลุ่มเป้าหมาย
  • กำหนดสิ่งพึงประสงค์: สร้าง/เสริมความเชื่อ กำหนด Cue
  • พัฒนา Narrative: น่าเชื่อ เชื่อมต่อ สม่ำเสมอ
  • กำหนดกิจกรรม: TA เห็นได้และเชื่อถือได้
AZ: Counter-PSYOP WL + Hidden Truth Campaign "ภาระของ WL"

MILDEC — การลวงทางทหาร

  • กำหนดเป้าหมายลวง: ผบ. ขตก. ผู้ตัดสินใจ
  • วิเคราะห์ช่องทาง: ผ่าน EW/กิจกรรม/เอกสาร
  • สร้างเรื่องราวลวง: น่าเชื่อ ส่งผ่านได้ วัดผลได้
  • ประสาน OPSEC: ปกปิดไม่ให้ ขตก. รู้ว่าถูกลวง
AZ: หลอกว่าจะขึ้นบกที่ TP-South → ปกปิดทิศจริง D-Day

OPSEC — รักษาความปลอดภัยปฏิบัติการ

  • กำหนด EEFI: ข้อมูลสำคัญยิ่งที่ต้องปกปิด
  • วิเคราะห์ช่องทาง ขตก.: สเปกตรัม SIGINT
  • วิเคราะห์สิ่งบอกเหตุ: อะไรที่ ขตก. รับรู้ได้
  • พัฒนามาตรการ OPSEC: ยุทธการ/ส่งกำลัง/EW
AZ: EMCON ก่อน D–3 · ปกปิดแผนขึ้นบก TALOS Main

EW — สงครามอิเล็กทรอนิกส์

  • ES: สเปกตรัม EM · SIGINT · TA
  • EA: กำหนดเป้าหมาย RSTA · Jamming
  • EP: ป้องกันไม่ให้ ขตก. ใช้คลื่นฝ่ายเรา
  • I&W: แจ้งเตือนการโจมตี EW ของ ขตก.
AZ: รบกวน/ดักฟัง C2 WL ใน TALOS · EMCON ฝ่าย AZ

CNO — ปฏิบัติการเครือข่ายคอมพิวเตอร์

  • CNA (Attack): โจมตีเครือข่าย C4 ขตก.
  • CND (Defense): ป้องกันระบบควบคุมฝ่ายเรา
  • CNE (Exploitation): รวบรวมข้อมูลจากเครือข่าย ขตก.
  • ประสาน EW/SIGINT: สนับสนุนกำหนดเป้าหมาย
AZ: DEFNET ป้องกัน Cyber WL · CNE ข้อมูล C2 WL

IO OBJ ๑ — Counter-PSYOP / StratComm

N7 IO จะดำเนินการ Convince + Inform ต่อ ประชาชน WL และประชาคมนานาชาติ (TA/มิติจิตวิทยา) โดยใช้ PSYOP + StratComm + Social Media — เพื่อ Create แรงกดดันทางการเมืองต่อรัฐบาล WL ให้ถอนกำลังออกจาก TALOS ตั้งแต่ D–30 จนถึง EOO

IO OBJ ๒ — MILDEC

N7 IO (MILDEC Cell) จะดำเนินการ Deceive ต่อ ผู้บัญชาการกองกำลัง WL ใน TALOS (TA/มิติสารสนเทศ-จิตวิทยา) โดยใช้ MILDEC Story + EW Signatures + Feint Operations — เพื่อ Enable ปฏิบัติการขึ้นบกหลัก D-Day ตั้งแต่ D–14 ถึง D+0

IO OBJ ๓ — OPSEC

ทุกหน่วยใน AZ TF จะดำเนินการ Protect ต่อ EEFI ฝ่ายเรา (TA/มิติสารสนเทศ) โดยใช้ OPSEC Measures + EMCON + EW-EP — เพื่อ Support ความได้เปรียบยุทธการ ตั้งแต่ D–30 ถึง D+5

IO OBJ ๔ — EW/CNO

N7 IO (EW/CNO Cell) จะดำเนินการ Disrupt ต่อ ระบบ C2 WL ใน TALOS (TA/มิติกายภาพ) โดยใช้ EW-EA + CNA (DEFNET) + SIGINT-ES — เพื่อ Enable ปฏิบัติการโจมตี D-Day ตั้งแต่ H–2 ถึง H+6
